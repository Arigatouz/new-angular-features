catchError must return an Observable

catchError must return an Observable. You must return EMPTY, of(value), or throwError(() => err). Returning undefined does NOT fail silently — RxJS throws: “You provided 'undefined' where a stream was expected.”

Every button fires the same real HttpClient request to a non-existent endpoint (always 404), then runs a different catchError handler.

💡 The network request is identical for all four — the 404 you see in DevTools is the raw failure. catchError runs on the client after that, so the difference shows up in the subscriber, not the Network tab. Each outcome is also logged to the console tagged with its strategy.
