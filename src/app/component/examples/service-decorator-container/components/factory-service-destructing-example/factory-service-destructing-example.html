Destructuring Comparison

One major benefit of the factory style is that it returns a plain object. This means methods are closures that don't depend on this, making them safe to destructure.

Factory Service (Safe)

@if (factoryStatus(); as status) {
{{ status.success ? 'Success' : 'Failed' }}
{{ status.message }}

Methods are closures. You can pull them out of the service object and they still work perfectly.

} 
/* ✅ This works */
const { increment } = factoryService;
increment();

Normal Service (Breaks)

@if (normalStatus(); as status) {
{{ status.success ? 'Unexpected' : 'Caught Error' }}
{{ status.message }}

Standard classes lose this context when destructured.

} 
/* ❌ This breaks */
const { increment } = normalService;
increment(); // Error!

Factory Service Style

This service is created using the factory property of the @Service decorator.

Count {{ factoryService.count() }}
Doubled {{ factoryService.doubled() }}

Implementation

@Service({
  factory: () => {
    const count = signal(0);
    return {
      increment: () => count.update((n) => n + 1),
    };
  },
})
